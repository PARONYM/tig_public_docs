# 権限・ロール
{: .no_toc }

CMS の操作権限は「ロール」で管理します。ユーザーやグループにロールを割り当てることで、見られる画面・行える操作が決まります。

## 目次
{: .no_toc .text-delta }

1. TOC
{:toc}

## ロールの考え方

- ユーザーには[ロール（権限）](roles.html)を割り当てます。
- 割り当てられるロールは、所属する[グループ](group.html)の種類によって変わります。
- 代表的なロールに **グループ管理者**（グループ内の管理ができる）と **一般ユーザ**（限定された操作）があります。

## 機能オプション

一般のグループでは、ロールに次の機能オプションを組み合わせて、利用できる機能を調整できます。

| オプション | 内容 |
|-----------|------|
| 動画編集（Tig編集） | コンテンツの編集機能を使える |
| ヒートマップ | ヒートマップ分析を見られる |
| アナリティクス | アナリティクス（視聴分析）を見られる |

## 操作の範囲

ロールは、操作の対象（コンテンツ・商品アイテム・分析・グループ・ユーザーなど）ごとに「閲覧・作成・編集・削除」の可否を持ちます。さらに、操作できる範囲は**自分が管理できるグループの範囲**に限られます。
